# 简历表述与项目亮点

这份文档不是讲代码细节，而是帮你把这个项目讲成一段更像“面试回答”的项目经历。

## 项目简介扩写版

WeHi 是一个面向企业 IM 场景的分布式消息系统项目，支持单聊、群聊、消息撤回、已读回执、在线状态同步、历史消息查询、断线重连和离线补偿。项目采用认证服务、业务 API 服务、实时网关服务三类职责拆分，通过 WebSocket 维护长连接通信，结合 Redis 管理在线状态、网关实例路由和高频临时状态，并基于 Kafka 构建事件总线，串联消息发送、实时广播、搜索索引和离线补偿链路，形成一套完整的实时消息系统骨架。

## 技术栈表述建议

`Go + Gin + MySQL + Redis + Kafka + WebSocket + Elasticsearch + Docker + OpenTelemetry + Kubernetes`

## 个人职责扩写版

### 1. 负责后端整体架构设计与服务拆分

将系统拆分为 `auth-service`、`api-service`、`realtime-service` 三类服务，分别承担认证鉴权、业务 API 和实时网关职责；同时按 `controller / service / repository / platform` 做分层设计，保证业务规则、数据访问和基础设施能力边界清晰，降低后续扩展和排障成本。

对应代码：

- `services/auth/main.go`
- `services/api/main.go`
- `services/realtime/main.go`
- `internal/routes/*.go`
- `internal/app/*`

### 2. 负责长连接通信与实时消息链路

基于 WebSocket 实现客户端长连接通信，设计连接鉴权、心跳检测、写超时、pong 保活和连接清理机制；将消息发送、送达、已读、撤回、输入态等状态变化通过事件驱动方式广播到在线客户端，确保在线会话具备低延迟交互体验。

对应代码：

- `internal/controllers/realtime_controller.go`
- `internal/realtime/hub.go`
- `services/realtime/main.go`
- `frontend/hooks/use-websocket.ts`

### 3. 负责 Redis 高频状态设计

使用 Redis 管理在线状态、网关实例路由、resume 恢复信息和登录会话等高频临时状态，把频繁波动的数据从 MySQL 中分离出来，减少数据库写压力，提升多实例实时链路的路由效率。

对应代码：

- `internal/app/auth/service.go`
- `internal/app/presence/service.go`
- `internal/platform/rabbit/rabbit.go`

### 4. 负责消息幂等、离线补偿与状态一致性

围绕 `client_msg_id` 设计消息发送幂等控制，避免弱网重试造成重复消息；通过 `sync_events + cursor` 方案实现断线后的增量补偿，支持客户端在重连后按游标拉取变更事件，追平消息列表、会话列表、已读状态等本地视图；同时通过 Kafka 事件流把送达、已读、索引和补偿链路串起来。

对应代码：

- `internal/app/chat/message_service.go`
- `internal/app/chat/sync.go`
- `internal/app/sync/service.go`
- `internal/app/repository/repository.go`
- `frontend/lib/chat-store.tsx`

### 5. 负责搜索、补偿与诊断相关链路

将消息和会话数据同时接入搜索索引链路，支持实时搜索与索引重建；基于 Docker + Kubernetes 完成服务注册发现与多服务部署；补充后台监控、消息旅程、会话一致性检查、AI 审计等诊断能力，使系统不只具备业务功能，也具备运维和排障能力。

对应代码：

- `internal/app/chat/search_service.go`
- `internal/platform/search/elasticsearch.go`
- `internal/routes/api.go`
- `internal/app/admin/*`

## 技术亮点扩写版

### 亮点 1：项目不是普通 CRUD，而是完整的实时消息系统骨架

它覆盖了：

- 长连接
- 缓存设计
- 状态同步
- 异步事件
- 离线补偿
- 搜索
- 可观测性

这比“管理后台 + 增删改查”更能体现系统设计能力。

### 亮点 2：消息生命周期和一致性处理比较完整

项目并不是只做到“发出去就结束”，而是把：

- accepted
- persisted
- delivered
- read
- recalled

这些状态串成了一条完整链路。

### 亮点 3：兼顾功能和工程化

除了主功能，还做了：

- migration
- smoke
- health check
- metrics
- trace
- 后台诊断

这说明你有工程意识，而不是只会堆接口。

- 宏观

- 有工程化
- 不会一上来就陷进细节

## 如果被问到 Kafka / Kubernetes

建议你重点讲两件事：

- Kafka：
  - 承接消息发送、送达、已读、索引、补偿等异步链路
  - 通过事件驱动把同步事务链和异步扩散链拆开
  - 更适合消费组扩展、顺序分区和高吞吐事件流
- Kubernetes：
  - 承接多服务部署、服务注册发现、健康检查和弹性伸缩
  - 配合 Docker 镜像完成交付和环境隔离
  - 提升服务扩展性与可维护性
